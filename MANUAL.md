## 1. 快速上手

1. 双击 `RedStudio.exe` 启动
2. 点击右上角 **⚙ 设置**
3. 在「供应商」区域添加你的 API（见第 2 节）
4. 点击顶部下拉框选择模型，开始对话

配置文件保存在 `C:\Users\你的用户名\.aistory\config.json`，历史记录在同目录 `history.json`。

## 2. 供应商与模型配置

### 添加云端 API（DeepSeek / Qwen / 豆包等）

设置 → 供应商 → **+ 添加供应商**

| 字段 | 说明 |
|------|------|
| 名称 | 随便起，如"深度求索" |
| 类型 | 选「OpenAI 兼容」 |
| Base URL | 供应商 API 地址，如 `https://api.deepseek.com` |
| API Key | 你的密钥 |

常用地址：
- DeepSeek：`https://api.deepseek.com`
- 通义千问：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- 豆包：`https://ark.volcengine.com/api/v3`
- Moonshot：`https://api.moonshot.cn/v1`

### 添加 Ollama 本地模型

| 字段 | 填写 |
|------|------|
| 类型 | 选「Ollama」 |
| Base URL | `http://localhost:11434`（默认） |
| API Key | 留空 |

添加后点击模型下拉旁边的 **刷新图标** 加载可用模型。

### 切换当前供应商

点击侧边栏顶部的供应商名称下拉，或在设置中调整顺序。

## 3. 对话基础操作

| 操作 | 方式 |
|------|------|
| 发送消息 | `Enter` 或点击 ↑ 按钮 |
| 换行 | `Shift + Enter` |
| 停止生成 | 点击底部「停止生成」按钮 |
| 复制 AI 回复 | 回复右下角复制图标 |
| 朗读 AI 回复 | 回复右下角朗读图标（🔊） |
| 新建对话 | 侧边栏顶部「+ 新建」按钮 |

### 系统提示词

消息框上方有可折叠的系统提示词输入区。每个对话独立设置。可从提示词库快速填入（点击 📚 图标）。

### Token 统计

输入框底部实时显示当前对话累计 token 消耗。侧边栏内嵌今日 token 仪表盘，按供应商分色统计。

## 4. 思考模式

**适用模型：** DeepSeek-R1、Qwen3（支持 `enable_thinking` 参数的模型）

点击输入框旁的 **💡 按钮**（紫色高亮 = 已开启）。

开启后 AI 会输出推理过程，以折叠块「思考过程」显示，正文在其下方。不支持思考模式的模型会忽略该参数。

## 5. 联网搜索

**前提：** 需要 Ollama 账号 API Key（免费注册：https://ollama.com）

### 配置

设置 → 联网搜索 → 填入 **Ollama API Key**

### 使用

点击输入框旁的 **🌐 按钮**（蓝色高亮 = 已开启）。

开启后每次发送消息会：
1. 自动以你的问题调用 Ollama 搜索 API（最多 5 条结果）
2. 将搜索摘要注入上下文
3. 连同你的问题发给 AI

**气泡显示的是原始问题**（搜索内容在后台注入，不影响显示）。

**适用所有模型**——DeepSeek、Qwen、Ollama 均可使用，不依赖模型本身支持工具调用。

## 6. 故事模式（角色扮演）

### 切换到故事模式

侧边栏顶部模式切换 → 选「故事」。

### 角色卡管理

**新建角色卡：** 点击侧边栏角色卡区域的「+ 新建」按钮。

字段说明：
| 字段 | 建议填写 |
|------|---------|
| 姓名 | 角色名（必填，用于识别） |
| 身份 / 称谓 | 如「猫耳女仆」「邻班同学」 |
| 外貌描述 | 外形关键词，逗号分隔 |
| 性格 | 关键词，如「傲娇、温柔」 |
| 语言风格 | 说话方式，如「自称「本小姐」」 |
| 爱好 | 可选 |
| 与主角关系 | 如「青梅竹马」 |
| 背景故事 | 角色设定，越详细越好 |
| 补充信息 | 禁忌话题、特殊规则等 |

**导入酒馆 PNG 卡：** 面板底部「导入 PNG」，选择 SillyTavern 导出的 PNG 文件，自动解析填入字段（需手动补全身高、爱好等细字段）。

### 使用角色卡

侧边栏「选择角色」下拉 → 选中后，角色设定自动拼入系统提示词，发送消息即开始角色扮演。

AI 回复中 `[角色名]` 前缀会渲染为彩色头像气泡。末尾的编号选项会渲染为可点击按钮。

## 7. 小说模式（互动写作）

### 切换到小说模式

侧边栏顶部模式切换 → 选「小说」。

### 角色库

点击侧边栏底部**角色库图标**（设置键左侧）打开角色库 overlay。

- **小说角色 / 聊天角色** 两个 Tab 分开管理
- 点击角色行直接选用，当前角色名显示在侧边栏标签
- 点「编辑」打开角色卡编辑面板；点「新建」新建角色
- 支持 **导出 JSON** / **导入 JSON**，方便在文本编辑器直接修改角色数据

女主角角色卡字段：姓名、出生日期、身高、身材、外貌、性格、语言风格、爱好、出身、当前状态、补充信息、**开场白**。

**支持 SillyTavern PNG 导入**，`first_mes` 自动映射为开场白字段。

### 开场白

角色卡「开场白」字段：预先写好第一段剧情正文（可含 `[CHOICES]` 选项块）。点击「▶ 开始新故事」后，开场白自动作为 AI 第一条回复插入，无需等待 AI 生成开头。

留空则由 AI 自动生成开头段落。

### 开始新故事

点击侧边栏「▶ 开始新故事」，弹出设定面板：

| 字段 | 说明 |
|------|------|
| 主角名称 | 主角的名字，注入系统提示词 |
| 故事方向 / 主角背景 | 剧情概述，可在此补充主角外貌、职业、性格等 |
| 叙事视角 | 第一人称（我）/ 第二人称（你）/ 第三人称（他） |
| 正文字数/轮 | 每次 AI 输出约多少字 |
| 好感度起始值 | 故事开始时的初始好感度（0–99） |
| 阶段划分 | 5 个阶段的名称和上限值，完全自定义 |

### 额外系统提示词

「系统提示词」栏的内容会追加到自动生成的提示词之后，可在此写故事骨架、输出规范等补充设定。

最终发给 AI 的系统提示词结构：
```
[自动生成基础提示词]

[故事方向 / 主角背景]

[女主角设定 — 角色名]

[输出格式与好感度规则]

[系统提示词栏手动输入内容]
```

### 章回选项与好感度

AI 回复末尾的 `[CHOICES]...[/CHOICES]` 块渲染为可点击选项按钮，每个选项附带好感度变化徽章（绿色 `+N` / 红色 `-N`）。自由输入时 AI 回复末尾追加 `[FAV:+N]` 标签自动更新好感度。

## 8. TTS 朗读

### 引擎选择（设置 → 朗读）

| 引擎 | 说明 |
|------|------|
| 小米 MiMo TTS | 在线，限时免费，4 个中文声线，需填 API Key |
| Edge TTS | 在线，微软神经网络，28 个中文声线，国内可能超时 |
| Windows SAPI | 离线，系统内置，无需联网 |

**MiMo API Key 获取：** https://platform.xiaomimimo.com（免费注册）

### 使用

点击 AI 回复右下角 🔊 按钮开始朗读。朗读中底部出现「⏹ 停止朗读」按钮，点击立即停止。再次点击朗读按钮也可停止。

### 语速调节

设置 → 朗读 → 语速（-10 最慢，0 正常，10 最快）。

## 9. 历史对话管理

- 侧边栏列出所有历史对话，点击切换
- 顶部搜索框实时过滤历史标题
- 每个历史条目右侧有删除按钮
- 对话标题自动取第一条用户消息前 24 字

## 10. 界面操作

| 操作 | 方式 |
|------|------|
| 折叠/展开侧边栏 | 交通灯右侧 ‹ 按钮 |
| 切换主题 | 右上角 🎨 按钮（5 种：亮色/护眼米/护眼绿/深蓝/纯黑） |
