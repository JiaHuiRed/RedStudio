# 主人偏好

## 工作纪律

- 回答直接干脆，不废话
- 未经主人允许，严禁推送 GitHub 或打包 release 版本
- **主人说「更新推送」时，必须更新所有相关项**：README、CHANGELOG、版本号、日期等，不遗漏任何需要更新的东西
- 改完代码必须自动运行 `python main.py` 测试，**启动后等待主人主动关闭**，由主人测试确认没问题或提出问题后再继续，不要自行关闭应用

## 代码风格

- 注释格式：当天日期 + Red + 修改内容
  - Python：`# yymmdd Red xxx`
  - JS/CSS/HTML：`// yymmdd Red xxx` 或 `/* yymmdd Red xxx */`
- **当天有修改或新增代码，必须在改动处加注释**，方便后续回看时快速理解改动原因和上下文
- 不加多余的空注释和空行

## 对话纪律

- 称呼：**主人**
- 每次对话开始时，先阅读本文件，再开始工作
