# 更新日志

本文件记录 Red Studio 的所有重要变更。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号规则：小改动 `x.x.patch`，中改动 `x.minor.0`，大改动 `major.0.0`。

---

## [3.3.5] - 2026-05-25

### 新增

- **快速预设**：侧边栏新增 2×2 预设按钮区（言情小说 / 奇幻RPG / 校园日常 / 武侠RPG），点击一键切换模式并注入对应场景提示词，激活预设高亮显示。
- **记忆压缩**：今日 Token 面板新增「压缩记忆」按钮，调用当前模型将旧消息生成摘要，替换 `state.messages` 前段（保留最近 4 条），聊天区插入可见分割线标记；复用现有流式管道，不新增 API 层。
- **通用 Toast 提示**（`showToast()`）：全局轻量提示函数，替代过去直接操作 DOM 的临时提示逻辑。

### 小说模式规则强化（核心痛点修复）

- **阶段行为禁止清单**：`buildNovelSystemPrompt()` 新增完整阶段对照表，每阶段列明明确的允许/禁止行为，当前阶段用 `◀ 当前` 标注，开局即告知 AI 全部边界规则。
- **每轮状态注入**（`buildNovelTurnReminder` / `injectNovelTurnReminder`）：每次发送请求前，将当前好感度、阶段、行为边界、格式要求打包为 system 消息插在用户输入之前（distance=1），紧邻上下文末尾，最大化 AI 注意力，防止多轮后规则遗忘。
- **`[CHOICES]` 缺失检测**：生成结束后若响应不含选项块，气泡底部显示红色内联警告 + 「重新生成」按钮，点击自动撤销上一条 AI 消息并以同一用户输入重新请求。

## [3.3.4] - 2026-05-23

### 新增

- **开场白字段**（`hc-opening`）：女主角角色卡新增「开场白」文本框，点击「开始新故事」后自动作为第一条 AI 回复插入，支持 `[CHOICES]` 选项块；SillyTavern PNG 导入时自动将 `first_mes` 映射到此字段。
- **角色库按钮**（侧边栏底部）：统一从侧边栏底部「角色库」图标按钮打开角色库 overlay，展示小说/聊天两类角色，支持 JSON 导出/导入；点击角色行直接选用，不再需要侧边栏下拉框。
- **新故事设定——主角名称**：新故事设定面板新增「主角名称」字段，随对话历史保存/恢复，注入系统提示词；故事方向文本框 placeholder 提示可补充主角背景。
- **叙事第一人称**：新故事设定新增「第一人称（我 / 主角名）」视角选项。
- **Textarea 自动高度**：故事方向输入框、系统提示词输入框随内容自动撑高（最大高度分别 200 px / 300 px）。

### 修改

- **简化设置面板**：移除「男主角设定（小说模式）」（9 个字段）和「小说模式提示词模板」两个设置组；主角信息改在每次新故事设定时填写，模板通过系统提示词栏追加。
- **小说系统提示词**：`buildNovelSystemPrompt()` 不再依赖 `config.novel_male_*` 字段，改用 `state.novelHeroName`；手动填写的系统提示词作为追加内容注入末尾（小说/RPG 模式均支持）。
- **角色卡下拉框移除**：侧边栏女主角区仅保留当前选中角色名标签 + 「开始新故事」按钮，角色选择统一在角色库 overlay 操作。

### 修复

- **关键 Bug：`const heroName` 重复声明**：`buildNovelSystemPrompt()` 内 `const heroName` 在同一作用域声明两次，导致 JS SyntaxError，app.js 整体崩溃（模型、历史、所有按钮全部失效）。已删除重复声明。

## [3.3.2] - 2026-05-22

### 重构

- **窗口缩放精简为单机制**：移除冗余的 `WH_MOUSE_LL` 低级别鼠标钩子和 JS 端边缘检测（`setupWindowResize`），缩放完全交由 Win32 `WM_NCHITTEST` 原生处理。彻底消除钩子与前端干扰导致的缩放抖动、失灵等问题。

### 修复

- **高 DPI 下四边/顶角无法拖拽缩放**：`WM_NCHITTEST` 中 `GetCursorPos` 返回物理像素，而 `frameGeometry` 返回 Qt 逻辑像素，DPI 缩放导致坐标错位 — 左/上边缘永不被检测，右/下边缘从窗口中间开始触发。改用 `QCursor.pos()` 统一为逻辑像素后修复。

## [3.3.0] - 2026-05-22

### 新增

- **故事模式角色卡**（`app.js` / `index.html`）：侧边栏新增角色选择区（仅故事模式显示），支持新建、编辑、删除女主角设定卡（姓名、身份/称谓、外貌描述、性格、语言风格、爱好、与主角关系、背景故事、补充信息）。选中角色卡后自动拼入系统提示词前缀 `【角色设定 — 名】`，与手动提示词共存。角色卡选择随对话历史保存/恢复。
- **小说模式女主角角色卡 CRUD**（`app.js`）：侧边栏下拉管理多个女主角角色卡（姓名、出生日期、身高、出身/所属、性格关键词、语言风格、爱好、代表作品、近期日程、补充信息），存入 `config.novel_heroines`，切换角色后自动重建小说系统提示词。
- **小说模式系统提示词自动构建**（`buildNovelSystemPrompt()`）：将用户 prompt 模板、男主角设定卡、女主角设定卡拼接成完整 system prompt，无需手动填写。
- **小说章回渲染**（`renderNovelChapter()`）：正文走 Markdown，末尾 `1. 2. 3. 4.` 格式选项渲染为可点击按钮。

## [3.2.2] - 2026-05-22

### 新增

- **侧边栏折叠**：标题栏新增折叠按钮，一键收起侧边栏，主内容区全屏展开，状态持久化。
- **重新生成按钮**：AI 回复底部操作栏新增重新生成（↺），触发去除最后一条消息并重新发送。

### 修复

- **Win11 高 DPI 缩放**：修复部分高分辨率屏幕下窗口抖动问题。
- **resize 截图覆盖防卡顿**：拖拽窗口边缘缩放时截图捕获导致的卡顿，改为节流处理。
- **覆层滚动闪烁**：移除所有 `backdrop-filter: blur()` 属性，消除 Chromium WebEngine 下滚动时的重合成闪烁（影响设置面板、提示词库、角色卡面板）。

## [3.1.0] - 2026-05-21

### 新增

- **窗口几何记忆**：退出时保存窗口位置与尺寸，下次启动恢复，默认 1440×1080。
