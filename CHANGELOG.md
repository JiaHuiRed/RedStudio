# 更新日志

本文件记录 Red Studio 的所有重要变更。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号规则：小改动 `x.x.patch`，中改动 `x.minor.0`，大改动 `major.0.0`。

---

## [3.6.3] - 2026-05-31

- **应用图标优化**：`icon.ico` 从单一分辨率（256px）升级为多分辨率 ICO（16/32/48/64/128/256），任务栏和桌面图标显示更清晰
- **代码维护**：清理 JRPG 模式死代码（`jrpgSavedNpcs`、`favDelta`）、更新过期注释

## [3.6.2] - 2026-05-30

### JRPG 模式

- **NPC 面板重设计**：P5R 协力者卡片风格，每个 NPC 卡片包含：头像+元素属性角标、名字+身份、性格标签（最多 3 个）、好感进度条+数值、Rank 等级（1-4）+阶段名
- **元素属性可视化**：NPC 元素以 emoji 角标显示在头像右下角，进度条颜色跟随元素色
- **清理死代码**：移除 `state.jrpgSavedNpcs`、`parseContentBlocks` 中的 `favDelta`；NPC 数量改读 config `npc_count`

### TTS

- **小米 MiMo TTS 超时修复**：长文本分段并行下载、顺序播放（每段 ≤300 字符，最多 5 路并发），避免单次请求超时，大幅降低长文本朗读等待时间

### UI

- **重命名弹窗**：替换原生 `window.prompt()` 为自定义 modal，标题栏显示"RedStudio"，毛玻璃背景，风格与 macOS 主题统一

## [3.6.1] - 2026-05-30

### 小说模式

- **好感度状态栏移除**：小说模式不再显示好感度进度条/阶段名，好感度完全由用户提示词定义、AI 在回复中自行输出
- **系统提示词精简**：`buildNovelSystemPrompt()` 不再自动注入好感度阶段规则（陌生人/相识/朋友/暧昧/恋人），避免与用户自定义 prompt 冲突；`buildNovelTurnReminder()` 不再注入每轮好感度核验
- **选项格式扩展**：`parseContentBlocks` 新增 `A. B. C. D.` 格式识别（兼容中文顿号 `A、B、C、D`），与原有 `[CHOICES]` 块和 `1. 2. 3. 4.` 并列
- **格式校验更新**：流式结束后检测选项时同时识别三种格式（`[CHOICES]` / A-D / 1-4），不再误报"本轮未生成选项"
- **选项按钮简化**：点击选项仅发送文字，好感度变化徽章（+N/-N）保留作为视觉提示，不再驱动内部状态变更

### UI

- **标题栏重构**：从 flex 布局改为 CSS grid（`auto 1fr auto`），交通灯和侧边栏按钮包入 `#titlebar-left`，流式徽章包入 `#titlebar-right`，标题在中间区域 `1fr` 真正居中，不再与交通灯重叠
- **标题栏高度**：48px → 36px，更接近 macOS 风格
- **交通灯/侧边栏按钮**：从绝对定位改为 flex 子元素，由 grid 布局控制位置

## [3.6.0] - 2026-05-28

### JRPG 模式（重大改版）

- **模式更名**：RPG 模式更名为 JRPG 模式，标签/UI 全部更新
- **NPC 好感系统**：5 个随机生成的女角色（名字/定位/性格/身材/外貌随机组合），每个独立好感度 0-100，4 阶段（陌生人→朋友→暧昧→恋人）
- **属性克制表**：10 种属性（火/水/地/风/冰/雷/木/光/暗/全），克制 2 倍伤害，被克制 0.5 倍
- **社交属性（德智体美）**：替换旧 5 属性，魅力直接影响好感度获取倍率
- **NPC 关系面板**：状态栏 👥 按钮展开，显示头像/名字/定位/好感进度条/阶段
- **NPC 保存到角色库**：悬停 NPC 行显示 💾 按钮，保存后下次新游戏可导入
- **升级通知**：等级提升时弹出 🎉 LEVEL UP 提示
- **战斗克制提示**：战斗选项显示 🟢（效果拔群）/🔴（效果不好）
- **模板驱动**：系统提示词改为模板化，内置"校园异世界"预置（世界设定+故事结构+NPC 池+属性表）
- **故事结构**：6 幕（序章→觉醒→集结→真相→低谷→决战→终章）
- **随机名字生成**：20 姓 × 20 名随机组合，确保每次游戏角色不同

### Bug 修复

- **旧存档兼容**：加载旧 RPG 存档时自动过滤已废弃的社交属性字段
- **清理死代码**：移除引用旧字段的 `buildRpgSystemPrompt` 函数

## [3.5.0] - 2026-05-27

### 架构统一

- **选择系统统一**：小说和 RPG 模式统一使用 `[CHOICES]` 块解析（`parseContentBlocks`），删除旧 `parseStoryContent()` 和两套独立选项渲染路径
- **统一交互**：两模式选项按钮共用 `.novel-choice-btn` CSS，点击均自动发送（`setTimeout(sendMessage, 300)`），不再填充输入框等待手动回车
- **选项类型检测**：`parseContentBlocks` 自动识别第二字段是数值（好感变化）还是字符串（RPG 类型如 combat/dialogue），两模式共用同一解析器
- **CSS 精简**：移除 `.story-choices` / `.story-choice-btn` 全套样式

### 小说模式

- **模板系统**：设置面板新增模板管理（增/删/改），模板存储用户自定义 prompt + 默认主角/POV/字数/好感开关；内置"言情宿舍"和"体育竞技"预置模板
- **动态好感阶段**：阶段表改为可增删行的 `renderNovelStages()`，告别硬编码 5 行；好感上限自动取最后阶段 cap，不再锁死 100
- **视角精简**：移除第二人称选项，保留第一/第三人称，默认第三人称
- **历史重命名**：对话历史条目新增重命名按钮（✎），支持 prompt 弹窗改名
- **打开动画修 bug**：`injectOpeningMessage()` 中 `renderNovelChapter` 参数传反，导致开场白 `[CHOICES]` 选项不渲染

### RPG 模式

- **角色提取独立**：RPG 的 NPC `[角色名]` 前缀提取抽为独立函数 `extractCharName()`，不再耦合 `parseStoryContent()`
- **选项自动发送**：`renderRpgChapter()` 点击选项后自动发送，与小说模式行为一致

### 杂项

- **应用图标**：换为去白底皮卡丘多分辨率 ICO（16/32/48/64/128/256），替换原精灵球图标

## [3.3.2] - 2026-05-22

### 重构

- **窗口缩放精简为单机制**：移除冗余的 `WH_MOUSE_LL` 低级别鼠标钩子和 JS 端边缘检测（`setupWindowResize`），缩放完全交由 Win32 `WM_NCHITTEST` 原生处理。彻底消除钩子与前端干扰导致的缩放抖动、失灵等问题。

### 修复

- **高 DPI 下四边/顶角无法拖拽缩放**：`WM_NCHITTEST` 中 `GetCursorPos` 返回物理像素，而 `frameGeometry` 返回 Qt 逻辑像素，DPI 缩放导致坐标错位 — 左/上边缘永不被检测，右/下边缘从窗口中间开始触发。改用 `QCursor.pos()` 统一为逻辑像素后修复。
